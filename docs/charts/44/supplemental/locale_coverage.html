This chart shows the coverage levels in this release. Totals are listed after the main chart.

Column Key

Default RegionThe default region for locale code, based on likely subtags
№ LocalesNote that the coverage of regional locales inherits from their parents.
Target LevelThe default target Coverage Level in CLDR. Particular organizations may have different target levels. Languages with high levels of coverage are marked with ‡, even though they are not tracked by the technical committee.
≟Indicates whether the CLDR Target is less than, equal to, or greater than the Computed Level.
Computed LevelComputed from the percentage values, taking the first level that meets a threshold (currently 🄼 99.5%, ⓜ 99.5%, ⓑ 100%).
ICUIndicates whether included in the current version of ICU
ConfirmedConfirmed items as a percentage of all supplied items. If low, the coverage can be improved by getting multiple organizations to confirm.
🄼%, ⓜ%, ⓑ%, ⓒ%Coverage at Levels: 🄼 = Modern, ⓜ = Moderate, ⓑ = Basic, ⓒ = Core. The percentage of items at that level and below is computed from confirmed_items/total_items. A high-level summary of the meaning of the coverage values is at Coverage Levels. The Core values are described on Core Data.
Missing FeaturesThese are not single items, but rather specific features, such as plural rules or unit grammar info. They are listed if missing at the computed level. For more information, see Missing Features
Example: ⓜ collation means this feature should be supported at a Moderate level.
  • Except for Core, these are not accounted for in the percent values.
  • The information needs to be provided in tickets, not through the Survey Tool.
TSVFiles:
  • locale-coverage.tsv — A version of this file, suitable for loading into a spreadsheet.
  • locale-missing.tsv — Missing items for the CLDR target locales.
  • locale-missing-summary.tsv — Summary of missing items for the CLDR target locales, by Section/Page/Header.
  • locale-missing-basic.tsv — Missing items that keep locales from reaching the Basic level.
  • locale-missing-counts.tsv — Counts of items per locale that are found, unconfirmed, or missing, at the target level. (Or at *basic, if there is no target level.)
